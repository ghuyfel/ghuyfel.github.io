What personal information do we collect from the people who use List Them All?
 
When using Talking Alarm, none of your Personally Identifiable Information (PII) is sent to us. We DO NOT collect any of the data that are stored or generated while using Talking Alarm. To generate the audio file that will be played in the alarm the text you would like the alarm to "speak" is sent directly to the Text-To-Speech server. It resides there the time of the generation of the audio file and deleted right after. For more information visit TTS_API.COM and TTSENGINE.COM
 
 
What permissions are required for Talking Alarm to run efficiently?
 
When you run Talking Alarm for the first time, the application will ask your permission to access the following:
 
  • Calendar
  • Location
  • Shared Files
  • Connect to BBM
  • Run Headless
 
Why does List Them All need to access them?
 
  • Location prenission is required to get the weather forecast of your location. If this permission is denied, Talking Alarm will not be able to set the shop location.
  • Calendar access is required to display yout upcoming events and populate the events widget. If this permission is denied, Talking Alarm will not be able to get and display your upcoming events.
  • Shared Files is required if you want to set your own ringtones for audio files located on your device or you SD Card. If this permission is denied, you will not be able to set personalized ringtones in Talking Alarm.
  • Run Headless is required for Talking Alarm to run headless. If this permission is denied Talking Alarm will not work at all.
  • Connect to BBM is required for updating your BBM status. If denied, Talking Alarm will not be able to update your BBM status.
